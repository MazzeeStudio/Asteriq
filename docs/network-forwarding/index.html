Network Forwarding

Network forwarding lets you use your HOTAS setup from a different PC on your local network. The machine with the physical controllers transmits input to another machine, which presents it as virtual joystick devices to Star Citizen.

Concepts​

RoleDescription
TX (Transmitter)The machine with the physical controllers. Reads input and broadcasts it to RX machines.
RX (Receiver)The machine running Star Citizen. Receives input from the TX machine and presents it to SC via vJoy.

Both machines must have Asteriq installed. The TX machine needs a full configuration with device assignments and mappings. The RX machine only needs vJoy installed.

Setting up the TX machine​

  1. Open the Settings tab and find the Network section.
  2. Enable the Network toggle.
  3. Select TX as the role.
  4. Note your pairing code — you will need this on the RX machine.
  5. Start Asteriq forwarding as normal from the Devices tab.

When an RX machine on the same network connects, it appears in the peers list. Toggle it on to begin transmitting to it.

Setting up the RX machine​

  1. Open the Settings tab, enable Network, and select RX as the role.
  2. Asteriq will discover TX machines on the local network automatically.
  3. Enter the pairing code from the TX machine when prompted to establish a trusted connection.
  4. Once connected, the RX machine receives the virtual joystick state and presents it to Star Citizen via vJoy.
tip

Enable Client Only Mode on the RX machine. This hides the Devices and Mappings tabs since they are not relevant when receiving input from a TX machine. See Client Only Mode.

Pairing​

The first time two machines connect, the RX machine must confirm the TX machine's pairing code. After pairing, the connection is trusted and reconnects automatically.

Connection status​

Both machines show connection status in the Network section of the Settings tab. The TX machine shows each connected RX peer. The RX machine shows the name of the TX machine it is connected to and a Disconnect button.

warning

Both machines must be on the same local network. Network forwarding does not work over the internet or across different subnets.
