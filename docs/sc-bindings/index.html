SC Bindings

The SC Bindings tab lets you load a Star Citizen control profile, assign inputs to SC actions across your devices, detect binding conflicts, and export a ready-to-use profile to your Star Citizen directory.

Layout​

The tab has two main areas:

  • Left — the action grid
  • Right — Device Order panel and Control Profiles panel

The action grid​

The grid lists every bindable SC action organised by category. Categories can be collapsed or expanded by clicking their header row.

Each row is one SC action. Columns represent your joystick devices — one column per vJoy slot. A cell shows the input currently bound to that action on that device.

Searching and filtering​

A search box at the top of the grid filters actions by name. The category filter narrows results to a specific action group.

Assigning a binding​

Click any cell to select it. The right panel shows an Assign button — click it and then move the physical control to detect the input. The binding is saved to the selected cell.

Conflict detection​

When two actions share the same physical input on the same device, Asteriq highlights both cells and lists the conflict in the right panel. Clicking a conflict entry scrolls the grid to the relevant action.

Device Order​

Star Citizen assigns joystick instance numbers (JS1, JS2, JS3…) to detected devices based on the order it sees them. The Device Order panel lets you control which vJoy slot maps to which SC joystick instance.

Each row shows a joystick slot (JS1, JS2…) and the vJoy device currently assigned to it. Click the assignment to change it.

Auto-detect — Asteriq reads the current DirectInput device order and automatically sets the instance assignments to match what Star Citizen will see. Run this after configuring vJoy devices and before exporting.

warning

If Device Order does not match what SC sees, your bindings will be applied to the wrong joystick in-game. Use Auto-detect or verify manually before exporting.

Control Profiles​

A Control Profile is the set of bindings you export to Star Citizen.

SC environment​

Select whether to export to your LIVE or PTU Star Citizen installation. Asteriq detects both installations if present.

Profile name​

Enter a name for the exported profile file. The file is written to your Star Citizen user directory. If a file with the same name already exists, it will be overwritten.

Exporting​

Click Export to write the profile to the Star Citizen directory. A status message confirms the export path. Launch Star Citizen and load the profile from the keybindings menu.

Importing​

Use the import dropdown to load an existing profile file from your Star Citizen directory. This populates the grid with the bindings from that file, which you can then modify and re-export.

Clear All / Reset Defaults​

Clear All — removes all bindings from the current profile.

Reset Defaults — restores the SC default bindings for the current schema.
