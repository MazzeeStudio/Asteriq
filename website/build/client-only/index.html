Client Only Mode

Client Only Mode is designed for RX machines in a network forwarding setup. When enabled, Asteriq hides the Devices and Mappings tabs, leaving only the SC Bindings and Settings tabs visible.

When to use it​

In a two-machine setup, the RX machine runs Star Citizen and receives virtual joystick input from the TX machine over the network. It does not need to manage physical controllers or configure axis mappings — those are handled on the TX machine.

Client Only Mode removes the tabs that are not relevant to the RX role, keeping the interface focused on what matters: Star Citizen bindings and connection settings.

Enabling Client Only Mode​

  1. Open the Settings tab.
  2. In the System panel, enable the Client Only Mode toggle.

The Devices and Mappings tabs disappear immediately. Disable the toggle at any time to restore them.

tip

Enable Client Only Mode after you have finished configuring the RX machine's network role in the Settings tab. The Settings tab remains accessible regardless of this setting.

What stays visible​

TabVisible in Client Only Mode
DevicesNo
MappingsNo
SC BindingsYes
SettingsYes

SC Bindings remains available so you can manage your Star Citizen control profile on the RX machine — assigning bindings to the virtual joystick slots that the TX machine populates via network forwarding.
