Why Asteriq?

One Application

No more juggling between multiple applications. Asteriq handles device assignment, input mapping, device hiding, and SC binding export in a single window.

Stable Device Identity

Physical devices keep their identity across sessions. No more bindings shifting when you unplug a USB cable or reboot. Devices are tracked by hardware ID, not port order.

Safe SC Integration

Asteriq reads SC's action definitions and can import existing mapping files. Exports are written as actionmaps.xml to your SC user folder, with a prompt before overwriting.

Getting Started

  1. 1
    Download and install

    Grab the MSI installer or portable ZIP from GitHub. No admin rights needed for the portable version.

  2. 2
    Install drivers

    On first launch, Asteriq checks for vJoy and HidHide. Follow the prompts to download and install. Both are optional but recommended.

  3. 3
    Assign devices

    Your sticks and throttles appear automatically. Drag to reorder, then use 1:1 Auto-Map for a quick passthrough setup.

  4. 4
    Tune mappings

    Open the Mappings tab to adjust curves, deadzone, and button modes. The live preview shows your physical input mapped through the response curve in real-time.

  5. 5
    Export SC bindings

    Switch to the SC Bindings tab, assign your actions, and click Export to SC. Load the exported profile in Star Citizen's keybind settings.

Device Management

Enumerate, order, hide, and visualise all your connected HOTAS hardware. Interactive silhouettes show your exact device layout with labelled controls.

Device Enumeration

All connected joysticks, throttles, pedals, and gamepads are detected automatically via SDL2. Each device shows its name, hardware ID (VID:PID), and connection status with a live indicator.

Disconnected devices remain in the list so you don't lose your setup when unplugging for transport.

Interactive Silhouettes

SVG-based device silhouettes render your exact hardware layout. Hover over a control to highlight it; click to select it for quick binding in the Mappings tab.

Lead-lines connect each physical control to its label. Silhouettes mirror automatically for left-hand variants. Currently available for Virpil and VKB hardware, with generic placeholders for other devices.

Live Input on Silhouettes

Silhouette lead-lines update in real-time as you interact with your hardware. Axis controls show a deflection bar, and buttons display their pressed or released state directly on the device image.

Useful for verifying hardware works correctly before creating mappings, or for identifying which physical control corresponds to which input.

Button Press Detection

When a button or hat is pressed, the corresponding lead-line on the silhouette highlights with a PRESSED indicator. This makes it easy to identify physical controls on unfamiliar hardware.

The Devices tab also provides quick actions: create 1:1 passthrough mappings, clear all mappings, remove disconnected devices, and toggle HidHide per device.

Mappings & Curves

Define how physical inputs translate to vJoy outputs. Shape axis response with interactive curves, set deadzone, configure button modes, and merge multiple inputs into one output.

Response Curves

Shape your stick response with four curve types. Linear for raw passthrough, S-curve for precision in the centre with faster edges, exponential for responsiveness, or custom with draggable control points.

A live preview shows your physical axis movement mapped through the curve in real-time, so you can feel the difference immediately.

Deadzone Control

Eliminate stick drift with precise deadzone adjustment. Centre mode creates a symmetric deadzone around the neutral point, which is ideal for joysticks. Throttle mode supports asymmetric deadzone for one-directional controls.

Drag the handles directly on the slider, or use preset buttons (0%, 2%, 5%, 10%) for quick adjustment. Supports axis inversion.

Button Modes

Four output modes for button mappings:

  • Normal: output mirrors input (pressed while held)
  • Toggle: each press flips state (on/off)
  • Pulse: brief activation on press (100 to 1000 ms)
  • Hold-to-Activate: only fires after holding for a duration (200 to 2000 ms)

Duration sliders let you fine-tune pulse and hold timing.

Keyboard Output

Map any physical button to a keyboard key with full modifier support. Press the key you want to bind and Asteriq detects it automatically, including Ctrl, Shift, Alt, and AltGr combinations.

Useful for mapping HOTAS buttons to keyboard shortcuts in games that don't support joystick input natively.

Mapping List

The left panel lists all mappings for the selected vJoy device, with M1 (Buttons) and M2 (Axes) category tabs. Each row shows the mapped output and any assigned modifiers.

Rows with active modifier keys display the modifier name alongside the button number. Use the vJoy device arrows at the top to switch between virtual devices.

Star Citizen Bindings

A complete binding grid for every SC action. Assign inputs by pressing buttons, set activation modes, resolve conflicts, and export a clean actionmaps.xml.

Binding Grid

Every Star Citizen action displayed in a scrollable grid with columns for keyboard, mouse, and each joystick device. Actions are organised by category (Flight, Weapons, Vehicle, etc.) with collapsible headers.

Search by action name, filter by category, or toggle "Bound only" to see just your assigned bindings.

Binding Details & Activation Modes

Select any cell to open the Binding Details panel. See action info, current binding, and set the activation mode with a segmented control:

  • Press: standard press/release (default)
  • Hold: hold button to activate
  • Double Tap: tap twice quickly
  • Triple Tap: tap three times
  • Delayed Press: long press to activate

Activation modes are stored per-binding and exported to SC's XML format.

Column Import

Click a joystick column header to open Column Actions. Import bindings from another saved Asteriq profile or from a Star Citizen XML file. Select the source profile and column, then import.

Useful for copying a known-good layout between profiles or migrating from an existing SC setup.

Conflict Detection

Asteriq highlights binding conflicts in real-time. Amber cells indicate shared bindings across devices (allowed but warned). Red cells block export until resolved.

Click a conflict cell to see all affected actions with navigation links. Jump directly to the conflicting binding to resolve it.

Search & Button Capture

Type to search actions by name. Use the button capture mode to press a physical button on your HOTAS. Asteriq filters the grid to show only actions bound to that exact input.

Combined with the "Bound only" toggle and category filter, you can quickly find and manage any binding in a list of 1000+ SC actions.

  • Noise: film grain texture
  • Scanline: horizontal CRT-style lines
  • Vignette: edge darkening for focus

All sliders update in real-time with instant visual feedback.

Network Forwarding: TX Mode

Keep your HOTAS connected to one machine and use it on any other computer on your network. The TX machine reads your physical devices and forwards the vJoy state to whichever RX machine you choose.

No disconnecting or re-connecting needed. Per-peer toggle switches let you select which machine receives input. Assign a physical HOTAS button as a TX Toggle to switch between machines without touching the keyboard.

Network Forwarding: RX Mode

The client (RX) machine receives vJoy snapshots from the master. Once a trusted master is found on the network, the connection is established automatically. Your local vJoy devices then mirror the master's output.

Devices and Mappings tabs lock to read-only while connected, preventing accidental changes. Use the Disconnect button to regain local control.

Auto-Updates

Asteriq checks for updates automatically on startup. When a new version is available, download and apply it with two clicks. The app extracts, swaps, and relaunches without manual intervention.

Toggle auto-check on or off, or trigger a manual check any time from the Settings panel.
