Mappings

The Mappings tab is where you configure how each physical input is transformed before it reaches the virtual device. Axis response curves, deadzones, button modes, and multi-axis merging are all configured here.

Layout​

The tab uses a three-panel layout:

  • Left — the bindings list for the selected vJoy device
  • Centre — device silhouette visualisation
  • Right — settings for the selected binding

Selecting a vJoy device​

Use the < > arrows at the top of the left panel to cycle between your vJoy devices. The bindings list updates to show the axes and buttons for the selected virtual device.

Bindings list​

Two side tabs switch the list between:

  • M1 — axis mappings (up to 8 axes per vJoy device)
  • M2 — button mappings

Each row represents one output on the virtual device. A row that has a source assigned shows the physical device and control it is mapped from. Click any row to open it in the settings panel on the right.

Axis mappings​

Selecting an axis row opens the axis mapping settings.

Source​

The source section shows which physical device and axis is mapped to this virtual output. Click the row in the bindings list and move the physical control to auto-detect it, or select manually from the device and control dropdowns.

Curve type​

Four response curve presets are available:

PresetBehaviour
LinearOutput matches input directly — 1:1 response
S-CurveReduced sensitivity near centre, increased toward extremes
ExponentialProgressively increasing sensitivity from centre outward
CustomFully manual — place control points on the curve editor

In Custom mode the curve editor opens in the centre panel. Drag control points to shape the response curve. A live input readout shows the raw and processed values as you move the physical control.

Deadzone​

The deadzone slider has handles at each end of the travel range. Drag a handle inward to create a dead region where small inputs are ignored.

Centre — enables a centre deadzone in addition to the end deadzones. Useful for axes that drift near their resting position.

Invert — flips the axis direction.

Merge mode​

Multiple physical axes can be merged onto a single virtual output. This is useful when rudder input comes from both a twist grip and a set of pedals.

Once more than one source is assigned, select how the values are combined:

ModeBehaviour
AVGAverages all input values
MAXUses the highest input value
MINUses the lowest input value
SUMAdds all values, clamped to the -1 to 1 range

Button mappings​

Selecting a button row opens the button mapping settings.

Source​

As with axes, move the physical button after clicking a row to auto-detect it, or select manually.

Button mode​

ModeBehaviour
NormalOutput mirrors the physical button — held while pressed
ToggleEach press flips the output between on and off
PulseOutput fires a brief signal on each press
Hold to ActivateOutput activates only after the button is held for a set duration

Pulse and Hold to Activate modes have a configurable duration in milliseconds.
