Asteriq User Guide

Asteriq is a unified HOTAS management application for Star Citizen. It replaces the need for separate tools like JoystickGremlin, the vJoy configuration GUI, and HidHide's own interface — everything is managed from one place.

System Requirements

RequirementDetails
OSWindows 10 or 11 (64-bit)
RuntimeNone — Asteriq is self-contained
DriversvJoy 2.x (optional but recommended)
HidHide (optional, needed to hide physical devices from SC)

vJoy and HidHide are the same drivers you may already have from JoystickGremlin setups. Asteriq replaces the GUI tools, not the drivers themselves.

Installation

  1. Download Asteriq.zip from the latest release.
  2. Extract the ZIP to any folder (e.g. C:\Asteriq).
  3. Run Asteriq.exe.

There is no installer — just extract and run. To uninstall, delete the folder.

Driver Setup

On first launch, Asteriq checks whether vJoy and HidHide are installed. If either is missing, a setup dialog appears.

Screenshot: Driver Setup dialog showing vJoy and HidHide status
  • vJoy is required for virtual device output. Without it, Asteriq can still visualise SC bindings, but mapping and forwarding are unavailable.
  • HidHide is optional. It hides your physical devices from Star Citizen so only the vJoy virtual devices are visible — preventing double-input problems.

You can re-open this dialog at any time from Settings > DRIVER SETUP.

First Launch

After driver setup, Asteriq opens with four tabs across the top:

TabPurpose
DevicesSee connected hardware and vJoy assignments
MappingsCreate axis curves, button maps, and output configurations
SC BindingsManage Star Citizen keybind XML files
SettingsProfiles, themes, network, updates

A profile dropdown in the title bar lets you switch between saved configurations.

Devices Tab

The Devices tab shows all connected hardware and how it maps to vJoy virtual devices.

Screenshot: Devices tab with physical device list, vJoy assignments, and silhouette

Physical Devices

The left panel lists every physical joystick, throttle, gamepad, and input device detected via SDL2. Each entry shows the device name, manufacturer, VID:PID, and a live connection indicator.

Press any button or move any axis on your hardware — the corresponding device entry will pulse briefly to confirm detection.

Virtual Devices (vJoy)

The assignment follows the device list order: first device → vJoy 1, second → vJoy 2, and so on. Up to 16 vJoy slots are available.

Device Ordering

Drag physical devices up and down to change their vJoy slot assignment. This order is saved in your active profile. Asteriq tracks devices by hardware identity, not USB port — swapping ports won't break your mapping.

Device Silhouettes

When a device has a matching silhouette map, a visual representation shows the physical layout with labelled controls and lead-lines. Hover to highlight, click to select.

Screenshot: Device silhouette with lead-line labels

Silhouettes are currently available for Virpil hardware. Other devices display a generic placeholder.

Hiding Devices (HidHide)

With HidHide installed, Asteriq can hide physical devices from other applications so Star Citizen only sees the vJoy virtual devices — preventing "double input" problems. Asteriq automatically whitelists itself.

Mappings Tab

The Mappings tab is where you define how physical inputs translate to vJoy outputs.

Screenshot: Mappings tab showing mapping list, curve editor, and silhouette

Creating a Mapping

  1. Select a vJoy device using the < > arrows at the top.
  2. Click Add Mapping (or double-click an empty row).
  3. The Mapping Dialog opens.
Screenshot: Mapping Dialog

Input detection: Move an axis or press a button — Asteriq auto-detects the device, input type, and index (15-second timeout).

Manual entry: Use dropdowns to choose the input device and specific control.

Output: Choose vJoy Axis, vJoy Button, vJoy POV, or Keyboard key.

Axis Mappings

Translate analog inputs (sticks, throttle sliders) to vJoy axis outputs with:

  • Curve — shape the response (see Curve Editor)
  • Deadzone — centred (joystick) or end-only (throttle)
  • Saturation — limit the output range
  • Inversion — reverse the axis direction

Multiple inputs can merge into one output: Average, Min, Max, or Sum.

Button Mappings

ModeBehaviour
NormalOutput mirrors input — pressed when held, released when let go
ToggleEach press flips the output state (on → off → on)
PulseBrief activation on press (100–1000 ms)
Hold-to-ActivateOutput activates only after holding for a duration (200–2000 ms)

Curve Editor

Shape axis response with an interactive graph. Drag control points to adjust, or use presets: Linear, S-Curve, Exponential. The preview shows real-time output as you move the physical axis.

Screenshot: Curve editor graph

TX Toggle Assignment

Designate a physical button as the TX TOGGLE — a hardware switch that toggles network forwarding on the master machine. The button row shows a "TX TOGGLE" badge when assigned.

SC Bindings Tab

Manage Star Citizen keybinding XML files — import existing layouts, assign new bindings, and export for SC to use.

Screenshot: SC Bindings tab showing binding grid

SC Installation Detection

Asteriq auto-detects LIVE, PTU, and EPTU installations. Select the environment from the dropdown — Asteriq reads defaultProfile.xml from SC's Data.p4k for the full action list.

Binding Grid

Rows = SC actions, columns = input sources (Mouse, Keyboard, vJoy). Filter by category or search by name.

To assign: click a cell, then press a button or move an axis. 5-second timeout.

Column Actions

Click a vJoy column header to select it. Actions appear:

ActionDescription
Import from ProfileLoad bindings from a saved profile or SC XML file
Clear ColumnRemove all bindings for this vJoy device
DeselectReturn to full grid view

Exporting to Star Citizen

  1. Set a profile name.
  2. Click EXPORT to generate actionmaps.xml.
  3. The file is saved to SC's bindings directory.
Read-Only Policy

Asteriq never modifies SC's existing binding files. It only writes new export files.

Conflict Detection

Amber = shared binding across multiple vJoy devices. Red = hard conflict (must resolve before export).

System Settings

  • Auto-load profile — load the last-used profile on startup
  • Close to tray — minimise to system tray instead of exiting
  • Tray icon type — throttle or joystick icon

Visual Settings

Font: size (0.8x–2.0x), family (Carbon or Consolas).

Theme: 12 colour themes — Midnight, Matrix, Amber, Ice, plus SC manufacturer themes (Drake, Aegis, Anvil, Argo, Crusader, Origin, MISC, RSI).

Background effects (sliders 0–100): Grid, Glow, Noise, Scanline, Vignette.

Network Forwarding

Enable network forwarding toggle, machine name, role selector (TX/RX), per-peer connection toggles, trust code management. See Network Forwarding Guide.

Updates

Version display, auto-check toggle, manual check button. When an update is available: DOWNLOAD → APPLY → auto-relaunch.

Network Forwarding Guide

Share input between two Asteriq machines on a LAN — one master (TX), one client (RX).

┌───────────────┐ TCP/LAN ┌───────────────┐ │ TX (Master) │──────────────▶│ RX (Client) │ │ │ │ │ │ Physical │ │ vJoy devices │ │ devices │ │ receive │ │ → Mappings │ │ snapshots │ │ → vJoy out │ │ from master │ │ → Snapshot │ │ │ │ to client │ │ SC reads │ │ │ │ local vJoy │ └───────────────┘ └───────────────┘

TX (Master) Setup

  1. Settings > Network — enable network forwarding.
  2. Set role to TX.
  3. A trust code is generated. Share it with the RX operator.
  4. Discovered RX peers appear with a toggle switch each.

RX (Client) Setup

  1. Settings > Network — enable network forwarding.
  2. Set role to RX.
  3. Wait for the TX machine to connect, or enter the trust code.
  4. Devices and Mappings tabs are locked while connected.

Connecting

Both machines must be on the same LAN. Discovery is automatic (UDP port 47191).

  • Click a peer's toggle to connect. The knob slides to the middle (connecting) then fully right (connected).
  • Click a different peer to switch — single tap, no disconnect step needed.
  • Click the connected peer's toggle to disconnect.
Firewall

Allow Asteriq through Windows Firewall on both machines (UDP 47191 for discovery, TCP for data).

Profiles

A profile stores your complete configuration: all mappings (axes, buttons, hats, curves, deadzones), device ordering, shift layers, silhouette selections, and the TX Toggle assignment.

Profiles auto-save on every change. Export as .json to share or back up. Import on another machine to use the same configuration.

Auto-Update

Asteriq checks GitHub for new releases on startup (if enabled) and on manual request.

  1. Check — polls the GitHub releases API.
  2. Download — fetches the update ZIP to a temporary location.
  3. Apply — replaces the application files and relaunches.

All your settings, profiles, and preferences are preserved — only the application binary is replaced.

Troubleshooting

Devices not detected

  • Check the device in Windows Device Manager.
  • Asteriq uses SDL2 — some very old or exotic devices may not be detected.
  • Try unplugging and reconnecting.

vJoy not working

  • Run Settings > DRIVER SETUP to verify installation.
  • Ensure at least one device is enabled in the vJoy Configuration utility.
  • Restart Asteriq after configuring vJoy.

Star Citizen sees double inputs

  • Install HidHide and hide physical devices.
  • Only vJoy devices should be visible to SC.
  • In SC's Options > Keybinding, verify only "vJoy" entries appear.

SC Bindings tab shows no actions

  • Ensure SC is detected (check the dropdown at the top).
  • Asteriq reads from SC's Data.p4k. Corrupted or incomplete installations may fail.
  • Run Asteriq.exe --scdetect to diagnose.

Update check fails

  • Ensure internet access.
  • GitHub's API has rate limits — wait a few minutes and retry.
  • Check logs in %APPDATA%\Asteriq\Logs\.

Application crashes on startup

  • Delete %APPDATA%\Asteriq\appsettings.json to reset settings.
  • Run Asteriq.exe --driver-setup to re-check drivers.
  • Check the latest log file for the error.
