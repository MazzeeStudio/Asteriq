What is Asteriq?

Running HOTAS in Star Citizen typically means juggling JoystickGremlin for input mapping, a separate vJoy configuration tool, and HidHide's own GUI. Asteriq replaces all of those with a single application. The vJoy and HidHide drivers are still installed, but you manage everything from Asteriq instead.

Input is handled via SDL2 — the same foundation JoystickGremlin uses — which tends to be more reliable than DirectInput. Device assignments are tracked by hardware identity rather than port order, so the configuration stays consistent regardless of how things are plugged in.

Features

Device Management

Each physical controller is assigned to a specific vJoy slot — that slot is what Star Citizen sees. Assignments are tied to the device's hardware identity, so nothing shifts if you replug cables in a different order.

If HidHide is installed, Asteriq can hide your physical devices from other applications whilst forwarding. This prevents SC from seeing both the real hardware and the virtual devices simultaneously.

Axis Mapping & Curves

Axes support Linear, S-Curve, Exponential, and fully Custom response curves. In custom mode you place control points manually and get a live input vs output readout as you move the stick.

Multiple physical axes can be merged onto a single virtual output — useful if rudder input is split across a twist grip and separate pedals. Average, maximum, minimum, or sum merge modes are all available. A symmetrical deadzone with quick presets rounds things out.

Star Citizen Bindings

The SC Bindings tab reads your current Star Citizen profile and displays every action alongside its bound input across all your joystick columns. You can reassign actions directly in Asteriq and export to actionmaps.xml when you're done.

Asteriq never modifies your existing SC config — it only writes to the export file. Conflict detection highlights actions that share the same input across multiple bindings so you can resolve them before they cause issues in-game.

Visual Themes

Choose from a set of manufacturer-inspired colour schemes — Drake, Aegis, Anvil, Crusader, Origin, Musashi, RSI, and others — or tune the palette manually. Background atmosphere controls (grid intensity, glow, noise, scanlines, vignette) let you dial in exactly the look you want.

Network Forwarding

Run Asteriq on one machine (TX) and receive the forwarded vJoy output on another (RX) over your local network. Useful if your HOTAS rig sits at a different PC from the one running Star Citizen — no extra software required on either end beyond Asteriq itself.

More of the App

Getting Started

  1. 1
    Download and extract

    Download Asteriq.zip using the button above and extract it to a permanent folder (for example C:\Tools\Asteriq\). Run Asteriq.exe from there. Windows may show an unknown publisher warning — click More info, then Run anyway.

  2. 2
    Install drivers if you need them

    Virtual joystick forwarding requires the vJoy 2.x driver. If you also want SC to only see the virtual devices and not your physical hardware, install HidHide as well. Asteriq runs without either, but most people will want at least vJoy.

  3. 3
    Assign your devices

    Open the Devices tab and assign each physical controller to a vJoy slot. That slot number is what Star Citizen will see as a joystick.

  4. 4
    Set up your mappings

    In the Mappings tab, dial in your axis curves and set button modes as needed.

  5. 5
    Start forwarding, then launch the game

    Click Start Forwarding and launch Star Citizen. Asteriq runs in the background and routes input through the virtual devices.
